# Mixtral MoE Forward 图解

> 参数: **3 token, hidden_size=10, 8 experts, top_k=2**

---

## 0. 代码全貌

```python
def forward(self, x):
    orig_shape = x.shape
    x_flat = x.view(-1, x.shape[-1])              # [B*S, dim]
    weights, indices = self.router(x_flat)          # Top-2 路由
    output = torch.zeros_like(x_flat)
    for i, expert in enumerate(self.experts):
        mask = (indices == i).any(dim=-1)
        if mask.any():
            token_ids = mask.nonzero(as_tuple=True)[0]
            expert_weights = weights[mask]
            idx_in_topk = (indices[mask] == i).float()
            w = (expert_weights * idx_in_topk).sum(dim=-1, keepdim=True)
            output[token_ids] += w * expert(x_flat[token_ids])
    return output.view(orig_shape)
```

## 2. Router (门控网络)

Router 是一个线性层: `nn.Linear(10, 8, bias=False)`, 将 10 维映射到 8 个专家的分数。

### 2.1 计算 logits

```
gate 权重矩阵: [10, 8]

每个 token 的 10 维向量 × gate → 8 个专家分数

              E0    E1    E2    E3    E4    E5    E6    E7
token0 logits: [0.5,  2.1,  0.3,  1.8,  0.2,  0.9,  0.4,  1.1]
token1 logits: [1.9,  0.4,  0.6,  0.3,  2.3,  0.7,  1.5,  0.1]
token2 logits: [0.3,  0.8,  1.7,  0.4,  0.6,  2.5,  0.2,  1.4]
```

### 2.2 Softmax → Top-2 → 重新归一化

```
token0 softmax: [0.05, 0.25, 0.04, 0.18, 0.04, 0.07, 0.05, 0.09]
  → topk(2): indices=[E1, E3], raw_weights=[0.25, 0.18]
  → 归一化:  weights=[0.58, 0.42]

token1 softmax: [0.18, 0.04, 0.05, 0.04, 0.27, 0.05, 0.12, 0.03]
  → topk(2): indices=[E4, E0], raw_weights=[0.27, 0.18]
  → 归一化:  weights=[0.60, 0.40]

token2 softmax: [0.03, 0.05, 0.12, 0.03, 0.04, 0.27, 0.03, 0.09]
  → topk(2): indices=[E5, E2], raw_weights=[0.27, 0.12]
  → 归一化:  weights=[0.69, 0.31]
```

### 2.3 Router 输出汇总

```
weights:                    indices:
┌──────────────────┐        ┌──────────┐
│ token0: [0.58, 0.42] │        │ [E1, E3] │
│ token1: [0.60, 0.40] │        │ [E4, E0] │
│ token2: [0.69, 0.31] │        │ [E5, E2] │
└──────────────────┘        └──────────┘

路由分配一览:
  token0 → E1(0.58) + E3(0.42)
  token1 → E4(0.60) + E0(0.40)
  token2 → E5(0.69) + E2(0.31)
```

## 6. 关键要点

| 要点 | 说明 |
|------|------|
| **稀疏激活** | 8 个专家只激活 2 个, 计算量 = 2/8 = 25% |
| **维度不变** | 输入 10 维 → 每个专家输出 10 维 → 加权和仍是 10 维 |
| **权重归一化** | Top-2 权重重新归一化使和为 1, 保证数值稳定 |
| **负载不均** | 本例 E6/E7 完全闲置, 需要 load balancing loss 缓解 |
| **逐专家循环** | 遍历 8 个专家, 每个专家批量处理选中它的 token |
