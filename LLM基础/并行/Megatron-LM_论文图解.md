# Megatron-LM 论文图解 (1909.08053)

- **论文**: Megatron-LM: Training Multi-Billion Parameter Language Models Using Model Parallelism
- **作者**: Mohammad Shoeybi 等 (NVIDIA, 2019)
- **核心贡献**: 提出层内张量并行 (Tensor Parallelism)，只需在 PyTorch 中插入几个通信原语，即可将 Transformer 模型切分到多张 GPU 上高效训练

```
    输入 X [batch, seq, H]
    │
    ▼
┌───────────────────────────────────┐
│          Layer Norm               │

                │
                ▼
┌───────────────────────────────────┐
│       Self-Attention              │
│                                   │
│   Q = X·Wq   K = X·Wk   V = X·Wv│  ← 3 个 GEMM
│         │         │         │     │
│         ▼         ▼         ▼     │
│      ┌──────────────────────┐     │
│      │ Attention(Q,K,V)     │     │

│      └──────────┬───────────┘     │
│                 ▼                 │

                │

## 4. MLP 切分 —— 论文最核心的洞察

### 4.1 为什么不能按行切 A？

```
方案一：按行切 A（✗ 不好）
═══════════════════════════

X = [X1, X2]    A = ┌ A1 ┐
                     └ A2 ┘

GPU0 算: X1·A1 ──┐
                  ├──→ 必须先求和 ──→ 再做 GeLU
GPU1 算: X2·A2 ──┘
                  ↑
           ┌──────────────────────────────────┐
           │ ✗ 这里需要一次 AllReduce 同步!    │
           │   因为 GeLU(a+b) ≠ GeLU(a)+GeLU(b)│
           │   非线性函数不能拆开算!             │
           └──────────────────────────────────┘
```

### 4.2 按列切 A（Megatron 的选择）

```
方案二：按列切 A（✓ Megatron）
═══════════════════════════════

A = [A1, A2]     按列切开

              ┌─── GPU0: X·A1 → GeLU(X·A1) = Y1
    输入 X ───┤
    (两卡都有) └─── GPU1: X·A2 → GeLU(X·A2) = Y2

              ↑
       ┌─────────────────────────────────┐
       │ ✓ 各算各的 GeLU，不需要通信!     │
       │   因为列切后各自独立               │
       └─────────────────────────────────┘
```

### 4.3 MLP 完整切分流程

```
MLP 完整切分 (2 张 GPU 为例)
════════════════════════════

矩阵 A [H, 4H] 按列切:  A1 [H, 2H]    A2 [H, 2H]
矩阵 B [4H, H] 按行切:  B1 [2H, H]    B2 [2H, H]

         GPU 0                              GPU 1
    ┌─────────────┐                    ┌─────────────┐
    │             │                    │             │
    │  X (完整)    │                    │  X (完整)    │
    │  │          │                    │  │          │
    │  ▼          │                    │  ▼          │
    │  X · A1     │                    │  X · A2     │
    │  [b,s,2H]   │                    │  [b,s,2H]   │
    │  │          │                    │  │          │
    │  ▼          │                    │  ▼          │
    │  GeLU       │                    │  GeLU       │
    │  │          │                    │  │          │
    │  ▼          │                    │  ▼          │
    │  Y1 = 结果  │                    │  Y2 = 结果  │
    │  │          │                    │  │          │
    │  ▼          │                    │  ▼          │
    │  Y1 · B1    │                    │  Y2 · B2    │
    │  [b,s,H]    │                    │  [b,s,H]    │
    │  │          │                    │  │          │
    └──┤──────────┘                    └──┤──────────┘
       │                                  │
       └──────────┬───────────────────────┘
                  ▼
            ╔═══════════╗
            ║ AllReduce ║  ← 整个 MLP 前向只需这 1 次通信!
            ║ (求和)     ║     Z = Y1·B1 + Y2·B2
            ╚═════╤═════╝
                  ▼
             Z [b,s,H]  → Dropout → + Residual
```

```

════════════════════════════════════════════

Wq, Wk, Wv 都按列切 (按 head 分)
Wo 按行切

         GPU 0 (head 0~7)                    GPU 1 (head 8~15)
    ┌────────────────────┐              ┌────────────────────┐
    │                    │              │                    │
    │  X (完整)           │              │  X (完整)           │
    │  │                 │              │  │                 │
    │  ├── X·Wq1 → Q1   │              │  ├── X·Wq2 → Q2   │
    │  ├── X·Wk1 → K1   │              │  ├── X·Wk2 → K2   │
    │  └── X·Wv1 → V1   │              │  └── X·Wv2 → V2   │
    │       │  │  │      │              │       │  │  │      │
    │       ▼  ▼  ▼      │              │       ▼  ▼  ▼      │
    │  ┌──────────────┐  │              │  ┌──────────────┐  │
    │  │ Attention     │  │              │  │ Attention     │  │
    │  │ head 0~7     │  │              │  │ head 8~15    │  │
    │  │ (完全本地!)   │  │              │  │ (完全本地!)   │  │
    │  └──────┬───────┘  │              │  └──────┬───────┘  │
    │         ▼          │              │         ▼          │
    │    Attn1 · Wo1     │              │    Attn2 · Wo2     │
    │    [b,s,H]         │              │    [b,s,H]         │

             │                                   │
             └──────────┬────────────────────────┘
                        ▼
                  ╔═══════════╗
                  ║ AllReduce ║  ← 整个 Attention 前向也只需 1 次!
                  ╚═════╤═════╝
                        ▼
                   → Dropout → + Residual

## 6. f 和 g 算子 —— 通信的秘密

```
f 算子和 g 算子
════════════════

它们成对出现，包裹住并行计算区域:

    ┌─────────────────────────────────────────────────────┐
    │                                                     │
    │              f              并行区域              g   │
    │              │    ┌───────────────────────┐    │    │
    │   输入 X ───→│───→│ GPU0: 算一半          │───→│───→ 输出 Z
    │              │    │ GPU1: 算另一半         │    │    │
    │              │    └───────────────────────┘    │    │
    │                                                     │
    └─────────────────────────────────────────────────────┘

    ┌─────────────────────────────────────────┐
    │  f 算子:                                │
    │    前向: identity (直接传过去,不通信)     │
    │    反向: AllReduce (聚合梯度)            │
    │                                         │
    │  g 算子:                                │
    │    前向: AllReduce (聚合结果)            │
    │    反向: identity (直接传过去,不通信)     │
    └─────────────────────────────────────────┘

    为什么这样设计？

    前向:  X ─identity─→ 并行计算 ─AllReduce─→ Z     (聚合结果)
    反向:  ∂X ←AllReduce── ∂并行 ──identity── ∂Z     (聚合梯度)

    ✓ 前向后向各 1 次 AllReduce，完美对称!
```

PyTorch 实现（真的只有几行）:

```python
class f(torch.autograd.Function):
    def forward(ctx, x):
        return x                    # 前向什么都不做
    def backward(ctx, gradient):
        all_reduce(gradient)        # 反向聚合梯度
        return gradient

class g(torch.autograd.Function):
    def forward(ctx, x):
        all_reduce(x)              # 前向聚合结果
        return x
    def backward(ctx, gradient):
        return gradient             # 反向什么都不做
```

## 7. 一个完整 Transformer 层的通信全景

```
一个 Transformer 层的 4 次通信
═══════════════════════════════

    输入 X (每张 GPU 都有完整副本)
    │
    ▼
┌─ Layer Norm (重复计算, 不通信) ─┐
│                                 │
│   f (前向=identity)             │
│   │                             │
│   ▼                             │
│   ┌─────────────────────────┐   │
│   │  Self-Attention 并行区域 │   │
│   │  Q,K,V 按列切, Wo 按行切 │   │
│   └─────────────┬───────────┘   │
│                 │               │
│   g (前向=AllReduce) ◀── 通信 ① │
│   │                             │
│   ▼                             │
│   + Residual + Dropout          │
│   │                             │
└───┤─────────────────────────────┘
    ▼
┌─ Layer Norm (重复计算, 不通信) ─┐
│                                 │
│   f (前向=identity)             │
│   │                             │
│   ▼                             │
│   ┌─────────────────────────┐   │
│   │    MLP 并行区域          │   │
│   │  A按列切, B按行切         │   │
│   └─────────────┬───────────┘   │
│                 │               │
│   g (前向=AllReduce) ◀── 通信 ② │
│   │                             │
│   ▼                             │
│   + Residual + Dropout          │
│                                 │
└─────────────────────────────────┘
    │
    ▼
  输出 (每张 GPU 都有完整副本)


  总计: 前向 2 次 AllReduce + 反向 2 次 AllReduce = 4 次通信/层
```

## 8. Embedding 层的优化

```
输入 Embedding
══════════════

词表 E [H × V] 按词表维度切:

    Token IDs ──→ GPU0: 查 E1 (词表前半) ──┐
                                            ├─→ AllReduce → 完整 embedding
                  GPU1: 查 E2 (词表后半) ──┘


输出 Embedding (logits) —— 关键优化!
════════════════════════════════════

朴素做法 (✗ 通信量太大):

    X ──→ GPU0: X·E1 = logits1 ──┐
                                  ├─→ AllGather → 完整 logits [b,s,50000]
          GPU1: X·E2 = logits2 ──┘                ↑
                                         通信量 = b×s×V ≈ 巨大!

Megatron 优化 (✓ 融合 cross-entropy):

    X ──→ GPU0: X·E1 → 本地 softmax+CE ──→ loss1 ──┐
                                                      ├─→ AllReduce → 总 loss
          GPU1: X·E2 → 本地 softmax+CE ──→ loss2 ──┘      ↑
                                                  通信量 = b×s ≈ 极小!

    通信量缩减: V/1 ≈ 50000 倍!
```
