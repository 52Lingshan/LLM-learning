# 可变分辨率图像的位置编码

## 核心挑战

```
固定分辨率位置编码：
┌─────────────────────────────────────────┐
│ 预训练时学习固定位置编码表              │
│                                         │
│ Position Embedding: [196, 768]          │
│ 对应 14×14 = 196 个 patch 位置          │
│                                         │
│ 问题：                                   │
│ • 224×224 图像 → 196 patches ✅         │
│ • 384×384 图像 → 196 patches？❌        │
│   (实际需要 27×27 = 729 个位置)         │
└─────────────────────────────────────────┘
```

**核心问题**：预训练的位置编码表是固定大小的，如何支持任意分辨率？

---

## 解决方案概览

| 方法 | 原理 | 优点 | 缺点 |
|------|------|------|------|
| **插值法** | 将预训练位置编码插值到目标尺寸 | 简单，保留预训练知识 | 插值可能失真 |
| **2D 分解** | 高度和宽度方向分别编码 | 灵活，泛化能力强 | 需要特殊设计 |
| **RoPE 2D** | 旋转位置编码的 2D 扩展 | 相对位置，泛化好 | 计算稍复杂 |
| **MoonViT 方法** | 插值 + 2D RoPE 结合 | 兼具两者优点 | 实现复杂 |

---

## 方法一：插值法

### 原理

```
预训练位置编码 (14×14)          插值到 (27×27)
┌───────────────┐            ┌─────────────────────┐
│ P1  P2  ... P14│            │ P1  P1.5 P2  ... P14 │
│ ...           │     →      │ ...                 │
│ P183 ...  P196│            │ P...  ...      P196 │
└───────────────┘            └─────────────────────┘
     196 个位置                    729 个位置
```

### 代码实现

```python
import torch
import torch.nn.functional as F

def interpolate_pos_embed(pos_embed, new_grid_size):
    """
    将预训练的位置编码插值到新的网格大小

    Args:
        pos_embed: [1, num_patches, dim] 预训练位置编码
        new_grid_size: 目标网格大小 (H, W)
    """
    # 分离 CLS token (如果有)
    cls_token = pos_embed[:, 0:1, :]  # [1, 1, dim]
    patch_pos = pos_embed[:, 1:, :]   # [1, num_patches, dim]

    # 获取原始网格大小
    num_patches = patch_pos.shape[1]
    orig_size = int(num_patches ** 0.5)  # 假设是正方形

    # reshape 为 2D 网格
    patch_pos = patch_pos.reshape(1, orig_size, orig_size, -1)  # [1, H, W, dim]
    patch_pos = patch_pos.permute(0, 3, 1, 2)  # [1, dim, H, W]

    # 双线性插值
    new_pos = F.interpolate(
        patch_pos,
        size=new_grid_size,
        mode='bicubic',  # 双三次插值，效果更好
        align_corners=False
    )

    # reshape 回序列格式
    new_pos = new_pos.permute(0, 2, 3, 1)  # [1, H, W, dim]
    new_pos = new_pos.reshape(1, -1, new_pos.shape[-1])  # [1, H*W, dim]

    # 拼接 CLS token
    new_pos = torch.cat([cls_token, new_pos], dim=1)

    return new_pos
```

### 问题

```
┌─────────────────────────────────────────┐
│ 插值法的问题：                           │
│                                         │
│ 1. 非正方形图像处理困难                  │
│    1920×1080 → 插值比例不一致            │
│                                         │
│ 2. 极端分辨率效果差                      │
│    14×14 插值到 100×100 失真严重         │
│                                         │
│ 3. 丢失位置编码的语义                    │
│    插值后的位置编码可能偏离预训练分布     │
└─────────────────────────────────────────┘
```

## 方法二：2D 分解位置编码

### 原理

将 2D 位置编码分解为高度和宽度两个方向：

```
传统方法：
Position(x, y) = Embed_2D[x, y]  # 需要预定义 H×W 大小

分解方法：
Position(x, y) = Embed_H[x] + Embed_W[y]  # 只需要 H + W 大小
```

### 图示

```
┌─────────────────────────────────────────────────────────────┐
│                    2D 分解位置编码                           │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   高度方向编码              宽度方向编码                     │
│   ┌───┐                    ┌─────────────────┐             │
│   │ H1│                    │ W1 W2 W3 W4 W5  │             │
│   │ H2│                    └─────────────────┘             │
│   │ H3│                           +                        │
│   │ H4│                          │
│   └───┘                    ┌─────────────────────────────┐ │
│        +                   │ (H1,W1) (H1,W2) ... (H1,W5) │ │
│                            │ (H2,W1) (H2,W2) ... (H2,W5) │ │
│   组合结果：               │ ...                         │ │
│   Position[i,j]            │ (H4,W1) (H4,W2) ... (H4,W5) │ │
│     = Embed_H[i]           └─────────────────────────────┘ │
│     + Embed_W[j]          │
│                                                             │
│   参数量：H + W (而非 H×W)                                  │
└─────────────────────────────────────────────────────────────┘
```

### 代码实现

```python
class Decomposed2DPosEmbed(nn.Module):
    def __init__(self, max_h=100, max_w=100, embed_dim=768):
        super().__init__()
        # 分别存储高度和宽度方向的位置编码
        self.pos_embed_h = nn.Parameter(torch.zeros(1, max_h, embed_dim))
        self.pos_embed_w = nn.Parameter(torch.zeros(1, max_w, embed_dim))

    def forward(self, H, W):
        """
        Args:
            H: 高度方向的 patch 数
            W: 宽度方向的 patch 数
        Returns:
            pos_embed: [1, H*W, embed_dim]
        """
        # 获取当前分辨率的位置编码
        pos_h = self.pos_embed_h[:, :H, :]  # [1, H, dim]
        pos_w = self.pos_embed_w[:, :W, :]  # [1, W, dim]

        # 广播相加
        # pos_h: [1, H, 1, dim] -> [1, H, W, dim]
        # pos_w: [1, 1, W, dim] -> [1, H, W, dim]
        pos_embed = pos_h[:, :, None, :] + pos_w[:, None, :, :]

        # 展平为序列
        pos_embed = pos_embed.reshape(1, H * W, -1)

        return pos_embed
```

### 优势

```
✅ 支持任意分辨率
✅ 参数量更少 (H + W vs H × W)
✅ 泛化能力强
✅ 非正方形图像处理自然
```

## 方法三：2D 旋转位置编码 (2D RoPE)

### RoPE 回顾

1D RoPE 通过旋转向量来编码位置：

```python
# 1D RoPE
def rope_1d(x, pos, dim):
    """
    x: [batch, seq_len, dim]
    pos: 位置索引
    """
    theta = 10000 ** (-2 * torch.arange(0, dim, 2) / dim)
    angles = pos * theta

    # 旋转
    x_rot = x * torch.cos(angles) + rotate_half(x) * torch.sin(angles)
    return x_rot
```

### 2D RoPE 扩展

```
┌─────────────────────────────────────────────────────────────┐
│                      2D RoPE 原理                           │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   将向量分成两部分，分别编码高度和宽度：                     │
│                                                             │
│   x = [x_1, x_2, ..., x_d]                                  │
│       ↓                                                     │
│   x = [x_h | x_w]  # 分成两半                               │
│       ↓                                                     │
│   x_h' = rotate(x_h, angle_h)  # 高度方向旋转               │
│   x_w' = rotate(x_w, angle_w)  # 宽度方向旋转               │
│       ↓                                                     │
│   x' = [x_h' | x_w']  # 拼接                                │
│                                                             │
│   旋转角度：                                                 │
│   angle_h = y * θ_h  (y 是高度位置)                         │
│   angle_w = x * θ_w  (x 是宽度位置)                         │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 代码实现

```python
class RoPE2D(nn.Module):
    def __init__(self, dim, max_h=100, max_w=100, base=10000):
        super().__init__()
        self.dim = dim
        self.half_dim = dim // 2

        # 预计算频率
        theta_h = base ** (-2 * torch.arange(0, self.half_dim // 2, 2) / dim)
        theta_w = base ** (-2 * torch.arange(0, self.half_dim // 2, 2) / dim)

        self.register_buffer('theta_h', theta_h)
        self.register_buffer('theta_w', theta_w)

    def forward(self, x, H, W):
        """
        Args:
            x: [batch, H*W, dim]
            H, W: 高度和宽度的 patch 数
        """
        batch_size = x.shape[0]

        # 生成位置索引
        pos_h = torch.arange(H, device=x.device)  # [H]
        pos_w = torch.arange(W, device=x.device)  # [W]

        # 创建 2D 位置网格
        pos_h = pos_h[:, None].expand(H, W)  # [H, W]
        pos_w = pos_w[None, :].expand(H, W)  # [H, W]

        # 计算旋转角度
        angles_h = pos_h.flatten()[:, None] * self.theta_h[None, :]  # [H*W, dim/4]
        angles_w = pos_w.flatten()[:, None] * self.theta_w[None, :]  # [H*W, dim/4]

        # 分割向量
        x_h, x_w = x.chunk(2, dim=-1)  # 各 [batch, H*W, dim/2]

        # 分别旋转
        x_h_rot = self._apply_rotary(x_h, angles_h)
        x_w_rot = self._apply_rotary(x_w, angles_w)

        # 拼接
        return torch.cat([x_h_rot, x_w_rot], dim=-1)

    def _apply_rotary(self, x, angles):
        """应用旋转"""
        x1, x2 = x.chunk(2, dim=-1)
        cos = torch.cos(angles)
        sin = torch.sin(angles)
        return torch.cat([x1 * cos - x2 * sin, x1 * sin + x2 * cos], dim=-1)
```

### 优势

```
✅ 相对位置编码：注意力计算时自动包含相对位置信息
✅ 长度外推：可以处理比预训练更长的序列
✅ 无需学习参数：完全基于数学计算
```

## MoonViT-3D 的方法：混合策略

### 架构

```
┌─────────────────────────────────────────────────────────────────┐
│              MoonViT-3D 位置编码策略                            │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   输入：Patch Embeddings [H*W, dim]                            │
│                    ↓                                            │
│   ┌─────────────────────────────────────────────────────────┐  │
│   │         Learnable2DInterpPosEmbDivided_fixed            │  │
│   │                                                         │  │
│   │  1. 可学习的绝对位置嵌入 (分解式)                        │  │
│   │     pos_embed_h: [max_H, dim]                           │  │
│   │     pos_embed_w: [max_W, dim]                           │  │
│   │                    ↓                                     │  │
│   │  2. 插值到当前分辨率                                     │  │
│   │     pos_h = interpolate(pos_embed_h, H)                 │  │
│   │     pos_w = interpolate(pos_embed_w, W)                 │  │
│   │                    ↓                                     │  │
│   │  3. 组合                                                 │  │
│   │     abs_pos = pos_h[:,:,None] + pos_w[:,None,:]         │  │
│   └─────────────────────────────────────────────────────────┘  │
│                    ↓                                            │
│   ┌─────────────────────────────────────────────────────────┐  │
│   │                    2D RoPE                               │  │
│   │                                                         │  │
│   │  在注意力计算时应用旋转位置编码                          │  │
│   │  提供相对位置信息                                        │  │
│   └─────────────────────────────────────────────────────────┘  │
│                    ↓                                            │
│   最终输出：绝对位置 + 相对位置                                │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 为什么结合两种方法？

```
绝对位置嵌入 + 2D RoPE = 最佳效果

┌─────────────────────────────────────────────────────────────┐
│                     优势互补                                 │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  绝对位置嵌入：                                              │
│  ✅ 提供每个 patch 的绝对空间位置                           │
│  ✅ 保留预训练的位置知识                                    │
│  ❌ 泛化到新分辨率需要插值                                  │
│                                                             │
│  2D RoPE：                                                  │
│  ✅ 提供相对位置信息                                        │
│  ✅ 更好的泛化能力                                          │
│  ✅ 支持任意分辨率                                          │
│  ❌ 缺乏绝对位置概念                                        │
│                                                             │
│  结合后：                                                    │
│  ✅ 绝对位置 + 相对位置                                     │
│  ✅ 预训练知识 + 泛化能力                                   │
│  ✅ 支持任意分辨率                                          │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 完整实现示例

```python
class MoonViTPositionEncoding(nn.Module):
    """MoonViT-3D 风格的位置编码"""

    def __init__(self, embed_dim=768, max_h=100, max_w=100, patch_size=14):
        super().__init__()
        self.patch_size = patch_size
        self.embed_dim = embed_dim

        # 可学习的分解式位置嵌入
        self.pos_embed_h = nn.Parameter(torch.zeros(1, max_h, embed_dim))
        self.pos_embed_w = nn.Parameter(torch.zeros(1, max_w, embed_dim))

        # 2D RoPE
        self.rope_2d = RoPE2D(embed_dim)

        # 初始化
        nn.init.trunc_normal_(self.pos_embed_h, std=0.02)
        nn.init.trunc_normal_(self.pos_embed_w, std=0.02)

    def forward(self, x, H, W):
        """
        Args:
            x: Patch embeddings [batch, H*W, dim]
            H, W: 高度和宽度的 patch 数
        """
        # 1. 绝对位置嵌入
        pos_h = F.interpolate(
            self.pos_embed_h.transpose(1, 2),  # [1, dim, max_H]
            size=H,
            mode='linear'
        ).transpose(1, 2)  # [1, H, dim]

        pos_w = F.interpolate(
            self.pos_embed_w.transpose(1, 2),
            size=W,
            mode='linear'
        ).transpose(1, 2)  # [1, W, dim]

        # 组合高度和宽度
        abs_pos = (pos_h[:, :, None, :] + pos_w[:, None, :, :])
        abs_pos = abs_pos.reshape(1, H * W, -1)  # [1, H*W, dim]

        # 2. 添加绝对位置嵌入
        x = x + abs_pos

        # 3. 应用 2D RoPE (在注意力层使用)
        # 这里返回处理后的 x 和 RoPE 函数
        return x, lambda q, k: self.rope_2d(q, H, W), lambda q, k: self.rope_2d(k, H, W)
```

## 总结

| 方法 | 核心思想 | 适用场景 |
|------|---------|---------|
| 插值法 | 缩放预训练位置编码 | 分辨率变化不大 |
| 2D 分解 | 高度宽度分别编码 | 任意分辨率 |
| 2D RoPE | 旋转编码相对位置 | 注意力机制 |
| **MoonViT 混合** | 绝对 + 相对位置 | 最佳效果 |

---

## 关键要点

1. **可变分辨率需要动态位置编码**
2. **分解式编码减少参数量并提高泛化**
3. **RoPE 提供相对位置信息**
4. **混合策略效果最佳**
