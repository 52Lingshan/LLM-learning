# Patch 切分详解

## 概述

**Patch 切分** 是 Vision Transformer (ViT) 将图像转换为 Transformer 可处理序列的核心操作。它将一张图像切成多个固定大小的小方块，每个方块称为一个 **Patch**。

---

## 直观理解

```
原始图像 (224×224×3)          Patch 切分              展平为序列
┌────────────────────┐      ┌──┬──┬──┬──┐         ┌─────────────────┐
│                    │      │P1│P2│P3│P4│         │ P1  P2  P3  P4  │
│                    │      ├──┼──┼──┼──┤    →    │ P5  P6  P7  P8  │
│      图像内容       │  →   │P5│P6│P7│P8│         │ P9  P10 P11 P12 │
│                    │      ├──┼──┼──┼──┤         │ P13 P14 P15 P16 │
│                    │      │P9│..│..│..│         │ ...             │
└────────────────────┘      └──┴──┴──┴──┘         └─────────────────┘
                              16×16 patches          序列长度: 196
```

---

## 详细流程

### 1. 图像到 Patch 的转换

```
输入图像: [H, W, C]  (高×宽×通道数)
    例如: [224, 224, 3]

Patch 大小: P × P
    例如: 16 × 16

Patch 数量: (H/P) × (W/P)
    例如: (224/16) × (224/16) = 14 × 14 = 196 个 Patches
```

### 2. 每个 Patch 的处理

```
单个 Patch:
┌─────────────┐
│ 16×16×3 =   │
│ 768 个像素  │
└─────────────┘
      ↓
  线性投影
      ↓
┌─────────────┐
│ 向量维度 D  │
│ 例如 768    │
└─────────────┘
```

### 3. 代码示意

```python
import torch
import torch.nn as nn

class PatchEmbedding(nn.Module):
    def __init__(self, img_size=224, patch_size=16, in_channels=3, embed_dim=768):
        super().__init__()
        self.img_size = img_size
        self.patch_size = patch_size
        self.num_patches = (img_size // patch_size) ** 2  # 196

        # 用卷积实现 Patch 切分 + 线性投影
        self.proj = nn.Conv2d(
            in_channels,
            embed_dim,
            kernel_size=patch_size,  # 16×16 卷积核
            stride=patch_size        # 步长=patch大小，无重叠
        )

    def forward(self, x):
        # x: [B, C, H, W] = [B, 3, 224, 224]
        x = self.proj(x)  # [B, embed_dim, H/P, W/P] = [B, 768, 14, 14]
        x = x.flatten(2)  # [B, embed_dim, num_patches] = [B, 768, 196]
        x = x.transpose(1, 2)  # [B, num_patches, embed_dim] = [B, 196, 768]
        return x
```

## 为什么需要 Patch 切分？

### Transformer 的输入要求

```
Transformer 处理的是「序列」数据：

NLP: 文本 → Token 序列
     "Hello world" → ["Hello", "world"] → [向量1, 向量2]

Vision: 图像 → Patch 序列
     图像 → Patches → [向量1, 向量2, ..., 向量N]
```

### 对比其他方法

| 方法 | 处理方式 | 问题 |
|------|---------|------|
| 像素展平 | 每个像素作为 token | 序列太长 (224×224=50176) |
| CNN 特征 | 提取特征图 | 不是纯 Transformer |
| **Patch 切分** | 固定大小切块 | ✅ 平衡序列长度与信息保留 |

## Patch 大小的影响

```
Patch 大小 = 14 (Kimi-VL 使用)
┌─────────────────────────────────┐
│ 224×224 图像 → 16×16 = 256 patches │
│ 每个 patch 包含更多信息          │
│ 序列较短，计算快                 │
└─────────────────────────────────┘

Patch 大小 = 32
┌─────────────────────────────────┐
│ 224×224 图像 → 7×7 = 49 patches  │
│ 每个 patch 包含更多像素          │
│ 序列最短，但细节丢失多           │
└─────────────────────────────────┘

Patch 大小 = 8
┌─────────────────────────────────┐
│ 224×224 图像 → 28×28 = 784 patches │
│ 每个 patch 更精细                │
│ 序列较长，计算量大               │
└─────────────────────────────────┘
```

## MoonViT-3D 的特殊处理

### 标准 ViT vs MoonViT-3D

```
标准 ViT:
┌─────────────────────────────────────────┐
│ 固定分辨率输入 → 固定数量 Patches        │
│                                         │
│ 224×224 图像 → 14×14 = 196 patches      │
│ 384×384 图像 → 必须缩放到 224×224        │
│               → 仍然是 196 patches       │
└─────────────────────────────────────────┘

MoonViT-3D:
┌─────────────────────────────────────────┐
│ 原生分辨率 → 动态数量 Patches            │
│                                         │
│ 224×224 图像 → 16×16 = 256 patches      │
│ 384×384 图像 → 27×27 = 729 patches      │
│ 1080×1920 图像 → 77×137 = 10549 patches │
│                                         │
│ Patch 大小固定 (14×14)，数量动态变化     │
└─────────────────────────────────────────┘
```

## 完整流程图

```
┌─────────────────────────────────────────────────────────────────┐
│                    Patch 切分完整流程                           │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  1. 输入图像                                                    │
│     ┌─────────────────────┐                                    │
│     │  [H, W, 3]          │  RGB 图像                          │
│     │  例如: [224,224,3]  │                                    │
│     └──────────┬──────────┘                                    │
│                ↓                                                │
│  2. Patch 切分 (通过 Conv2d)                                   │
│     ┌─────────────────────┐                                    │
│     │  kernel=patch_size  │  无重叠切分                        │
│     │  stride=patch_size  │  同时进行线性投影                  │
│     └──────────┬──────────┘                                    │
│                ↓                                                │
│  3. Patch Embeddings                                           │
│     ┌─────────────────────┐                                    │
│     │  [N, embed_dim]     │  N = (H/P) × (W/P)                 │
│     │  例如: [196, 768]   │  每个 patch 一个向量               │
│     └──────────┬──────────┘                                    │
│                ↓                                                │
│  4. 添加位置编码                                               │
│     ┌─────────────────────┐                                    │
│     │  + Position Embed   │  让模型知道 patch 的空间位置       │
│     └──────────┬──────────┘                                    │
│                ↓                                                │
│  5. 添加 [CLS] Token (可选)                                    │
│     ┌─────────────────────┐                                    │
│     │  [CLS] + Patches    │  用于分类任务                      │
│     └──────────┬──────────┘                                    │
│                ↓                                                │
│  6. 输入 Transformer Encoder                                   │
│     ┌─────────────────────┐                                    │
│     │  多层 Self-Attention│                                    │
│     └─────────────────────┘                                    │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 总结

| 概念 | 说明 |
|------|------|
| **Patch** | 图像的小方块，如 16×16 像素 |
| **Patch 切分** | 将图像切成多个 Patch |
| **Patch Embedding** | 每个 Patch 映射为一个向量 |
| **目的** | 让 Transformer 能处理图像 |
| **MoonViT 创新** | 支持动态 Patch 数量（原生分辨率）|
